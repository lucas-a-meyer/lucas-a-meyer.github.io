Posting automatically from my blog to LinkedIn

Author

Lucas A. Meyer

Published

June 28, 2022

Nobody reads my blog.

It’s not that I really care. It’s just that for some content, a blog post works a lot better than LinkedIn.

To my knowledge, there are three ways to post technical content on LinkedIn:

  • You can write an article
  • You can create a training
  • You can post an external link

The problem of writing an article on LinkedIn is that the editor is good for text pieces but unfriendly for technical content, especially for code and source control. In addition, the editor requires the writer to edit online all the time. In practice, I mostly copy and paste.

The problem of creating a training is that I think it’s a lot of work. I never tried, and maybe it’s easier than I think it is, but just getting started seems like a lot.

Instead of using LinkedIn, a very popular option is to use Medium or, more specifically, “Towards Data Science” (TDS), a subset of Medium articles focused on data science. I frequently find articles I like in TDS, but it’s somewhat paywalled. You get just a few free articles for free. For authors, the advantages are that TDS is widely promoted in search results and that authors get paid. Not enough to quit a job and write for TDS, but enough to buy a couple books per year. Other options are newsletters, but it starts to feel too much like a job.

The LinkedIn algorithm seems to punish posts that only point users to an external article. I can understand that: this type of post is unlikely to generate any engagement inside of LinkedIn: people just “click out”.

In any case, the problem I wanted to solve was to be able to write more technical posts from time to time, but still be able to post those to LinkedIn automatically. To write technical posts in my blog, I started using #Quarto and loving it (I’m putting together some material about that). I just spent some time on the weekend figuring out how to configure Quarto to post automatically to LinkedIn. I still needed to solve the problem on how to push these to LinkedIn in a somewhat good way.

So I wrote some code that scans my Quarto files and uses the LinkedIn API to post them.

When I write a blog post (technical or non-technical), I can choose to automatically post it to LinkedIn by simply using a YAML field.

This solves a large number of problems for me. With that, I now do all my writing in VS Code. I don’t need to use Buffer anymore. I have version control. If the post is too big, my tool will just cut the post short and link to the remainder.

As always, the proof of the pudding is in the eating, so let’s see how that goes.
